# App Store Submission Materials

This folder contains text and references for submitting GitSyncMarks-Mobile to app stores (Play Store, F-Droid, etc.).

## Contents

| File | Purpose | Max length |
|------|---------|------------|
| `short-description.txt` | Play Store short description | 80 chars |
| `full-description.txt` | Play Store / F-Droid full description | 4000 chars (Play) |

## Screenshots

Screenshots are generated by `flutter test --update-goldens test/screenshot_test.dart` and saved to:

```
metadata/en-US/images/
├── phoneScreenshots/     # Phone (Play Store, F-Droid)
│   ├── 1_bookmarks.png
│   ├── 2_empty_state.png
│   └── 3_settings.png
├── iphoneScreenshots/
├── ipadScreenshots/
├── macbookScreenshots/
└── flathubScreenshots/   # Flathub (Linux)
```

Use `metadata/en-US/images/phoneScreenshots/` for Play Store and F-Droid.

## Play Store

1. **Short description**: Copy from `short-description.txt` (80 chars max)
2. **Full description**: Copy from `full-description.txt` (4000 chars max)
3. **Screenshots**: Use `metadata/en-US/images/phoneScreenshots/*.png`
4. **Category**: Productivity or Books & Reference

## F-Droid

The app includes fastlane-compatible metadata for F-Droid. See `metadata/en-US/`:

- `short_description.txt`, `full_description.txt` — descriptions (auto-used by F-Droid)
- `images/icon.png` (512x512), `images/phoneScreenshots/*.png`
- `changelogs/{versionCode}.txt` — What's New per release

For submitting to F-Droid, see [fdroid/README.md](../fdroid/README.md).
